# Trained checkpoint

`maps_stage2_s1-2025_s2-2025_best_ssim.pt` (94 MB) — the paper's primary
MAPS generator: Stage-1 seed 2025 + Stage-2 seed 2025, `best_ssim` EMA
weights (UNetG, in_ch=6, base 80, 24.5 M parameters; offsets ±{3, 9, 15};
`pareto4` loss preset). This is the fixed-Stage-1 protocol checkpoint behind
Table 1 (Δφ = 0.00122 ± 0.00026 at `tri_mean`, deployment parity) and the
`s1_2025_s2_2025` cell of the initialization matrix (Table S2).

Usage:

```bash
python infer_triaxis.py \
    --checkpoint checkpoints/maps_stage2_s1-2025_s2-2025_best_ssim.pt \
    --volume_path data/BB_1000c_f32.bin \
    --cube_origin 861 372 372 --cube_size 128 256 256 \
    --agg tri_mean --parity
```

The file stores only the deployed EMA weights (`ema_state_dict.shadow`) plus
a `meta` record; the full training state (discriminator, optimizers) is not
included and is regenerated by retraining (`train_stage1.py` →
`train_stage2.py`).
